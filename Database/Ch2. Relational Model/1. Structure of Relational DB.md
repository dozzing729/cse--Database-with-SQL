# 1. Structure of Relational DataBases

## 기본 구조 / 용어

- **table**: 관계형 db는 table들의 집합체로, 각 table 마다 고유한 이름을 갖는다. 보통 한 행이 각 값들 사이의 관계를 나타내고, 테이블은 이러한 관계들의 집합이므로

  table의 개념은 수학의 relation 개념과 연관지을 수 있다. 그래서 이 모델 이름이 relational db라고 한다.
  
- **tuple**: table의 각 행

- **attributes**: table의 각 열. 속성은 보통 atomic이어야 한다.


## Keys

- **superkey**: 한 relation에서 한 tuple을 유일하게 식별할 수 있도록 해주는 속성 혹은 속성들의 집합이다.
- **candidate key**: 최소한의 superkey 
- **primary key**: relation 내에서 각 tuple들을 구별하기 위한 수단으로, db 설계자에 의해 candidate key중에서 선택된다.
  - **natural key**: 실제로 어떤 개체가 갖는 속성이 pk가 됐을 때 이를 natural key라고 부른다. 예를들어 사람의 주민등록번호가 pk가 된 경우 이에 해당한다.
  - **surrogate key**: 어떤 개체의 실제 속성은 아니지만 pk설정을 위해 인위적으로 추가한 속성을 의미한다. 
- **foreign key**: relation schema r1이 갖고 있는 속성들 중에 또다른 relation schema r2의 primary key가 되는 속성을 의미하고, 이러한 속성을 r1으로부터 r2를 참조하는 foreign key라고 한다.

  여기서 r1은 **referencing relation**, r2는 referenced relation이라고 한다.
  

## Database Schema

- **database schema**: db의 논리적 구조를 의미한다. 마치 프로그래밍 언어에서의 type definition과 같다.
- **database instance**: database schema에 실제로 저장된 데이터 값. 마치 프로그래밍 언어에서 변수의 값과 같다.
